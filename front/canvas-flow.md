# Canvas Flow — умови роботи канвасу

**Перевіряй цей файл щоразу при внесенні змін до канвасу.**

---

## Позиціонування

- `.land__canvas` — займає максимальну доступну ширину і висоту
- Розміри `.land__canvas` задаються в **CSS** (`width: 100%`, `height: 100%`), **не через JS**
- **Char:** по вертикалі центрується відносно `.land__canvas` (wrapperHeight); по горизонталі — offsetX від лівого краю (з breakpoints)

---

## Розміри canvas

- Задаються **брейкпоінтами з конфігу** (`canvasBreakpoints` з main.js)
- послідовність пріорітету брейкпоінтів згідно порядку їх передавання в масив, більше індекс = більше пріорітет
- Формат: `{ maxWidth, width, height }` або `{ maxWidth, width, height, isWrapperFill?: boolean }`
- **isWrapperFill:** якщо `true` — розмір canvas береться з wrapper (`.land__canvas`: offsetWidth, offsetHeight)


---

## Фон канвасу

- Фон завжди на всю площу канвасу: ширина = ширина канвасу, висота = висота канвасу
- Пропорційне масштабування відносно розміру canvas зі збереженням aspect ratio root-розмірів
- **Перемикання фонів:** коли ширина канвасу досягає root ширини картинки + 50px, перемикатися на картинку шириною меншою за попередню, але більшою за наступну

## Coins

- **Розміри:** 134×172px (з конфігу)
- **Шлях картинок:** `img/canvas/coin/` (static.png для static; frames: frame-1.png, frame-2.png, … для fade-out)
- **Стани:** `static` — кадр static.png з папки coin; `fade-out` — один цикл по кадрах frame-1, frame-2, потім коін стає прозорим (visible: false)
- **Позиціонування:** в ряд, відносно char
  - Перший коін — `offsetRight` px вправо від char (breakpoints); `offsetRightDefault` — fallback у конфігу
  - Кожен наступний — `gapBetween` px праворуч від попереднього (breakpoints)
  - Кастом: `items[i].gapBetweenLeft` — відступ зліва для коіна i; `items[i].gapBetweenRight` — відступ справа (впливає на наступний коін). У `breakpoints` можна `itemGaps: { index: { gapBetweenLeft, gapBetweenRight } }` — перевизначення по брейкпоінтах
  - По вертикалі — центрування як у char (wrapperHeight)
- **API:** `setCoinFadeOut(coinIndex)` — запустити fade-out для коіна з індексом `coinIndex`

---

## Barrier

- **Розміри:** 171×112px (з конфігу)
- **Шлях картинок:** `img/canvas/barrier/` (frames: frame-1.png … frame-6.png)
- **Стани:** `hide` — не видимий; `fade-in` — анімація появи; `static` — статичний кадр (frame 6)
- **Позиціонування:** barrier[i] центрується над coin[i], offsetAbove над монетою (breakpoints)
- **Тригер:** коли coin[i] закінчує fade-out, barrier[i] автоматично запускає fade-in

---

## Cars

- **Розміри варіантів:** car-1 168×342, car-2 170×384 (з конфігу)
- **Шлях картинок:** `img/canvas/car/car-1.png`, `car-2.png`
- **Візуал:** випадковий (car-1 або car-2) при кожному старті drive
- **Стани:** running (дефолтний), fade-in
- **Позиціонування:** над кожним coin[i] по горизонталі (центр машини = центр коіна), по вертикалі зверху за межами канвасу
- **Running:** машини їдуть вертикально зверху вниз, виїжджають за край, миттєво повертаються. Інтервал 4–10 с, max 2 одночасно глобально, max 1 машина в running на один coin slot, min 1 с між стартами. При `pendingJumpStart` нові не запускаються. Під час jumping — прискорення (множник з конфігу)
- **Fade-in:** при спрацюванні barrier — якщо машина в running, створюється копія для fade-in, оригінал завершує і видаляється; якщо idle, машина сама їде. Під'їзд до barrier, зупинка за 20px до нього, залишається на місці. max 1 fade-in машина на один coin slot
- **Перед jumping:** чекати завершення всіх running машин; не запускати нові running
- **Перед стрибком на коін N:** якщо для slot N є running машина — чекати її проїзду (`pendingJumpForCoinIndex`). Для slot N не спавнити нові running. Коли машина проїде — запустити стрибок
- **Fade-in при jumping:** min 2, max 3 машини з 4; max 2 підряд. Комбінації: [0,1], [1,2], [2,3], [0,2], [0,3], [1,3], [0,1,3], [0,2,3]
- **Порядок малювання:** Background → Coins → Cars → Barriers → Char

## Конфіг

- Всі дані для рендеру беруться з конфігу, що передається з `main.js` в `chicken-canvas.js`
- В конфігу повинна бути можливість передавати брейкпоінти і root для фону картинки, щоб примусово змінити його за потреби

---

## Resize / орієнтація

- При кожному resize чи зміні орієнтації: перераховувати всі розміри canvas та перезапускати його
- Цей функціонал в окремій функції, щоб можна було примусово її викликати за потреби
